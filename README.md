# مشروع التفريغ الصوتي المحلي (Parakeet TDT STT)

هذا المشروع يوفر بيئة متكاملة وسريعة لتشغيل نموذج **Parakeet TDT 0.6B v3** (بصيغة ONNX) محلياً. 

[المصدر](https://huggingface.co/istupakov/parakeet-tdt-0.6b-v3-onnx)

تم تصميم المشروع ليعمل بكفاءة عالية على المعالج (CPU)، مع استخراج نصوص وترجمات دقيقة التوقيت بصيغة `SRT` متوافقة مع معايير الترجمة المرئية. كما يتفوق على نموذج وايسبر.

---

## المميزات الأساسية
- **تحميل مباشر وذكي:** سكربت التحميل يجلب الملفات المباشرة من Hugging Face بدون الحاجة لمكتبات خارجية أو تسجيل دخول، مع دعم **شريط التقدم** و**استئناف التحميل** في حال انقطاع الاتصال.
- **تفريغ دقيق (TXT & SRT):** يقوم باستخراج النص الخام وملف ترجمة `SRT` ذو توقيتات مبنية بذكاء بناءً على علامات الترقيم والوقفات الطبيعية للمتحدث.
- **معالجة الصوتيات الآلية:** يدعم الملفات الصوتية الأحادية (Mono) والمزدوجة (Stereo) حيث يتم دمج القنوات تلقائياً لتفادي أي أخطاء.

---

## المتطلبات الأساسية
- أداة **uv** لإدارة بيئات بايثون (اختياري).
- ملف صوتي بصيغة `.wav`.

---

## طريقة التثبيت والتشغيل

### 1. إعداد البيئة (لأول مرة)
إذا لم تكن قد هيأت المشروع بعد، قم بفتح الطرفية في مجلد المشروع ونفذ:
```bash
uv init --python 3.14
uv add "onnx-asr[cpu]" 
```

إذا كان لديك بطاقة رسوميات من NVIDIA وأردت تسريع التفريغ بشكل كبير، يمكنك تغيير الاعتماديات لدعم كرت الشاشة بتنفيذ الأمر: `uv add "onnx-asr[gpu]"`.

### 2. تحميل النموذج
قم بتشغيل سكربت التحميل لجلب أوزان النموذج محلياً (حوالي 3 جيجابايت). السكربت يدعم الاستئناف، فإذا توقف التحميل يمكنك إعادة تشغيله بآمان:
```bash
uv run download.py
```
*(سيتم حفظ النموذج في مجلد `local_model`)*

### 3. تشغيل التفريغ الصوتي
1. ضع الملف الصوتي الخاص بك في المجلد الرئيسي وقم بتسميته `test.wav` (أو قم بتعديل الاسم داخل ملف `run.py`).
2. نفذ أمر التشغيل:
```bash
uv run run.py
```
